In order to process your continuous integration server's build reports, you must provide Build Watch with the URL of your server dashboard's RSS feed.

Note: RSS URLs are case-sensitive and should begin with the protocol, such as "http://".

CruiseControl.rb

You can find the URL of your dashboard's RSS feed by clicking on the RSS icon in your browser's location bar. The URL should end in "projects.rss". For example:

http://builder.com/projects.rss

CruiseControl.NET

The URL of your dashboard's RSS feed is not visibly displayed through the web interface. However, you can create it by replacing "ViewFarmReport.aspx" in your dashboard's URL with "XmlServerReport.aspx".

For example, if your dashboard's URL is:

http://builder.com/ViewFarmReport.aspx

then the RSS feed's URL will be:

http://builder.com/XmlServerReport.aspx

You can confirm this by entering the URL in your web browser before trying to add it to Build Watch.

CruiseControl for Java

CruiseControl for Java provides a number of RSS feeds. Build Watch needs to be supplied with the feed that's compatible with CCTray. The link for this feed is labeled "CCtray" and is listed on the right side of the CruiseControl dashboard, in the "Tools" section. The URL will end in "cctray.xml". For example:

http://builder.com/dashboard/cctray.xml

Note: CruiseControl servers have two dashboards. Make sure you're looking at the dashboard with the word "dashboard" in the URL, as in the example above.

Hudson

Hudson provides a number of RSS feeds. Build Watch needs to be supplied with the feed that's compatible with CCTray. This URL is simply your server's dashboard URL with "cc.xml" appended to the end. For example, if your Hudson server's dashboard URL is:

http://builder.com:8080

Then the RSS feed you should provide to Build Watch will be:

http://builder.com:8080/cc.xml
